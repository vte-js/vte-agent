# VTE Agent 架构插拔化重构方案

## 目标

将 VSCode 宿主能力抽离为可插拔的 HostAdapter 接口，使 VTE Agent 核心引擎可运行在多种宿主环境（VSCode、Web、Electron、CLI、JetBrains）。

## 当前架构分析

### 已有的框架无关层（✓ 可复用）
```
src/core/types.ts      — 接口定义（ToolDefinition, ContextManager, AgentEvent 等）
src/core/engine.ts     — 引擎核心（394行，框架无关版本，但未被使用）
src/tools/*.ts         — 工具实现（Node.js fs/path，不依赖 vscode）
src/context/manager.ts — 文件操作（Node.js fs）
src/agent/complexity.ts — 任务复杂度分析
src/agent/token-tracker.ts — Token 统计
src/shared/protocol.ts — 共享类型
```

### VSCode 耦合层（需要抽象）
```
src/vscode/extension.ts — 扩展激活、命令注册、状态栏
src/vscode/panel.ts     — WebviewViewProvider、postMessage 消息路由
src/vscode/lsp/*        — LSP 集成（vscode.workspace.fs, vscode.executeCommand）
src/agent/engine.ts     — 引擎实现（含 ShadowGit、权限、任务强制等扩展功能）
```

### 消息通信层（需要抽象）
```
webview/src/composables/useVsCode.ts — acquireVsCodeApi() + postMessage
```

### 核心接口：HostAdapter

```typescript
// src/host/types.ts

export interface HostAdapter {
  // ── 标识 ──
  readonly id: string           // 'vscode' | 'web' | 'cli' | 'electron'
  readonly name: string         // 显示名称

  // ── 文件系统 ──
  fs: HostFileSystem

  // ── 工作区 ──
  workspace: HostWorkspace

  // ── UI 交互 ──
  ui: HostUI

  // ── 消息通信 ──
  messaging: HostMessaging

  // ── 扩展能力（可选）──
  lsp?: HostLSP
  shell?: HostShell
  git?: HostGit
}

export interface HostFileSystem {
  readFile(path: string): Promise<string>
  writeFile(path: string, content: string): Promise<void>
  exists(path: string): Promise<boolean>
  stat(path: string): Promise<{ type: 'file' | 'directory'; size: number }>
  mkdir(path: string): Promise<void>
  readdir(path: string): Promise<string[]>
  delete(path: string): Promise<void>
  watch?(path: string, callback: (event: 'create' | 'change' | 'delete') => void): Disposable
}

export interface HostWorkspace {
  getRoot(): string | undefined
  getFolders(): Array<{ name: string; uri: string }>
  onDidChangeWorkspaceFolders?(callback: (event: any) => void): Disposable
}

export interface HostUI {
  // 对话框
  showInfo(message: string): Promise<void>
  showWarning(message: string): Promise<void>
  showError(message: string): Promise<void>
  confirm(message: string): Promise<boolean>

  // 选择器
  showQuickPick(items: Array<{ label: string; description?: string }>, options?: { multi?: boolean }): Promise<string[] | undefined>
  showInputBox(prompt: string, options?: { defaultValue?: string }): Promise<string | undefined>
  showOpenDialog(options: { canSelectFiles?: boolean; canSelectFolders?: boolean; filters?: Record<string, string[]> }): Promise<string[] | undefined>

  // Toast 通知
  toast(level: 'info' | 'success' | 'warning' | 'error', message: string, options?: { duration?: number }): void

  // 进度
  withProgress<T>(title: string, task: (progress: (message: string) => void) => Promise<T>): Promise<T>
}

export interface HostMessaging {
  // 向前端发送消息
  postMessage(msg: any): void

  // 监听前端消息
  onMessage(handler: (msg: any) => void): Disposable

  // 信号
  signalReady?(): void
}

export interface HostLSP {
  // LSP 能力（VSCode 独有，其他宿主可选实现）
  gotoDefinition?(uri: string, line: number, character: number): Promise<any[]>
  findReferences?(uri: string, line: number, character: number): Promise<any[]>
  hover?(uri: string, line: number, character: number): Promise<any>
  getDocumentSymbols?(uri: string): Promise<any[]>
  executeCommand?(command: string, ...args: any[]): Promise<any>
}

export interface HostShell {
  execute(command: string, options?: { cwd?: string; timeout?: number }): Promise<{ stdout: string; stderr: string; exitCode: number }>
}

export interface HostGit {
  getBranch(): Promise<string>
  getStatus(): Promise<string>
  diff(args?: string): Promise<string>
  log(args?: string): Promise<string>
}

export interface Disposable {
  dispose(): void
}
```

### VSCode HostAdapter 实现

```typescript
// src/host/vscode.ts

import * as vscode from 'vscode'
import { HostAdapter, HostFileSystem, HostWorkspace, HostUI, HostMessaging } from './types'

export class VSCodeHostAdapter implements HostAdapter {
  readonly id = 'vscode'
  readonly name = 'Visual Studio Code'

  fs: HostFileSystem
  workspace: HostWorkspace
  ui: HostUI
  messaging: HostMessaging
  lsp?: HostLSP
  shell?: HostShell
  git?: HostGit

  constructor(private extensionUri: vscode.Uri) {
    this.fs = new VSCodeFileSystem()
    this.workspace = new VSCodeWorkspace()
    this.ui = new VSCodeUI()
    this.messaging = new VSCodeMessaging()
    this.lsp = new VSCodeLSP()
    this.shell = new VSCodeShell()
    this.git = new VSCodeGit()
  }
}
```

import { HostAdapter } from '../host/types'

export class AgentEngine {
  private host: HostAdapter

  constructor(host: HostAdapter, config: RuntimeConfig) {
    this.host = host
    // ...
  }

  // 文件操作通过 host.fs
  async readFile(path: string) {
    return this.host.fs.readFile(path)
  }

  // UI 交互通过 host.ui
  async askUser(question: string) {
    return this.host.ui.showInputBox(question)
  }

  // 消息通过 host.messaging
  sendMessage(msg: any) {
    this.host.messaging.postMessage(msg)
  }
}

### 前端消息层改造

```typescript
// webview/src/composables/useHost.ts（替代 useVsCode.ts）

export interface HostTransport {
  postMessage(msg: any): void
  onMessage(handler: (msg: any) => void): void
  signalReady?(): void
}

// VSCode 实现
export function createVsCodeTransport(): HostTransport {
  const vscode = acquireVsCodeApi()
  return {
    postMessage: (msg) => vscode.postMessage(JSON.parse(JSON.stringify(msg))),
    onMessage: (handler) => window.addEventListener('message', (e) => handler(e.data)),
    signalReady: () => vscode.postMessage({ type: 'ready' }),
  }
}

// WebSocket 实现（Web/Electron）
export function createWebSocketTransport(url: string): HostTransport {
  const ws = new WebSocket(url)
  return {
    postMessage: (msg) => ws.send(JSON.stringify(msg)),
    onMessage: (handler) => ws.onmessage = (e) => handler(JSON.parse(e.data)),
  }
}

// useHost composable（替代 useVsCode）
export function useHost() {
  const transport = inject<HostTransport>('hostTransport')
  return {
    send: (msg: any) => transport.postMessage(JSON.parse(JSON.stringify(msg))),
    onMessage: (handler: (msg: any) => void) => transport.onMessage(handler),
    signalReady: () => transport.signalReady?.(),
  }
}
```

## 实施步骤（分阶段）

### Phase 1：定义接口 + VSCode 适配器（最小改动）
1. 创建 `src/host/types.ts` — 定义 HostAdapter 接口
2. 创建 `src/host/vscode.ts` — VSCode 适配器实现
3. 修改 `panel.ts` — 使用 VSCodeHostAdapter
4. 修改 `engine.ts` — 接收 HostAdapter 参数
5. 修改 `useVsCode.ts` → `useHost.ts` — 抽象消息层
6. 验证现有功能不受影响

### Phase 2：解耦工具层
1. 工具不再直接 import fs/path，通过 HostAdapter 访问
2. bash 工具通过 `host.shell.execute()` 执行
3. 文件工具通过 `host.fs` 操作

### Phase 3：解耦 LSP 层
1. LSP 功能作为 HostAdapter 的可选能力
2. 非 VSCode 宿主可以不实现 LSP，或实现自己的 LSP 客户端

### Phase 4：新宿主实现
1. Web 独立应用（WebSocket 传输 + 浏览器文件 API）
2. CLI 终端（stdin/stdout 传输 + Node.js fs）
3. Electron 桌面应用（IPC 传输 + Node.js fs）
4. JetBrains IDE 插件（JetBrains API）

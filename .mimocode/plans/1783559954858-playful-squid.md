# Checkpoint + Shadow Git 完善计划

## 目标
在现有 JSON 快照基础上，增加 Shadow Git 支持，实现更成熟的 checkpoint 系统。

## 架构设计

### 双轨存储
```
.vte/
├── checkpoints/           ← JSON 快照（当前方案，保留）
│   └── {id}.json
└── shadow-git/            ← Shadow Git 仓库（新增）
    └── .git/
```

- **JSON 快照**：保存完整状态（对话+文件+任务），快速恢复
- **Shadow Git**：只保存文件 diff，支持 `git diff` 查看变更

### 工作流
```
AI 编辑文件 → 自动 git add + commit（shadow-git）
用户保存 checkpoint → 创建 JSON 快照 + git tag
用户恢复 → 从 JSON 恢复状态 + git checkout 文件
LLM 查看变更 → git diff between commits
```

## 实现方案

### 1. Shadow Git 管理器
**新文件**: `src/agent/shadow-git.ts`

```typescript
class ShadowGit {
  private repoPath: string  // .vte/shadow-git
  
  init()                    // git init（如果不存在）
  trackFile(path: string)   // git add + commit
  commit(message: string)   // git commit
  diff(commit1?, commit2?)  // git diff
  log(n?: number)           // git log --oneline
  checkout(commit: string)  // git checkout
  tag(name: string)         // git tag
  restoreFile(path: string, commit: string) // 恢复单个文件
}
```

### 2. Engine 集成
**文件**: `src/agent/engine.ts`

- 构造函数中初始化 `ShadowGit`
- `createCheckpoint()` → 同时创建 JSON 快照 + git tag
- `restoreCheckpoint()` → 从 JSON 恢复 + git checkout
- 工具调用后自动 `trackFile()` 追踪修改

### 3. Checkpoint 工具扩展
**文件**: `src/agent/checkpoint-tools.ts`

新增工具：
- `checkpoint_diff`：查看两个 checkpoint 之间的文件差异
- `checkpoint_log`：查看 checkpoint 历史（含 diff 统计）

### 4. 类型定义
**文件**: `src/shared/types.ts`

Checkpoint 接口新增：
```typescript
interface Checkpoint {
  // ... 现有字段
  gitCommit?: string  // 对应的 git commit hash
  gitTag?: string     // git tag 名称
}
```

## Shadow Git 命令映射

| 操作 | Git 命令 |
|------|----------|
| 初始化 | `git init` |
| 追踪文件 | `git add {path}` |
| 提交 | `git commit -m "{message}"` |
| 查看差异 | `git diff {commit1} {commit2}` |
| 查看历史 | `git log --oneline -n {n}` |
| 恢复文件 | `git checkout {commit} -- {path}` |
| 创建标签 | `git tag {name} {commit}` |
| 按标签恢复 | `git checkout {tag}` |

## 验证方式

1. AI 编辑文件 → 检查 `.vte/shadow-git/` 有 commit
2. 保存 checkpoint → 检查 JSON + git tag 同时创建
3. 调用 `checkpoint_diff` → 验证返回 unified diff
4. 恢复 checkpoint → 验证文件内容回滚
5. 重启扩展 → 验证 shadow-git 保留

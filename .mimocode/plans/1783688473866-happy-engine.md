# Plan: Fine-Grained Permission Control System

## Goal
Implement a permission control system for code mode that restricts LLM tool usage and prompts user authorization when needed. Users can configure permission levels and choose to allow once, always allow, or deny each tool execution.

## Permission Categories

| Category | Tools | Risk Level |
|----------|-------|------------|
| **File Read** | `read`, `list`, `grep`, `glob` | Low |
| **File Write** | `edit`, `write` | High |
| **Terminal** | `bash` | High |
| **Git** | `git` (read-only commands) | Low |
| **Diagnostics** | `diagnostics` | Low |
| **Web** | `webfetch` | Medium |
| **Task** | `task_create`, `task_update`, `task_list`, `task_delete` | Low |
| **Checkpoint** | `checkpoint_*` | Low |

## Permission Levels

| Level | Behavior |
|-------|----------|
| `allow` | Always allow, no prompt |
| `ask` | Prompt user each time |
| `deny` | Always deny, block execution |

## Architecture

```
LLM requests tool call
  → engine.executeToolCalls()
  → Check permission for tool category
  → If 'allow' → execute immediately
  → If 'deny' → return error result, skip execution
  → If 'ask' → send { type: 'permission:request', tool, args } to webview
    → Webview shows AuthorizationDialog
    → User selects: allow_once | always_allow | deny
    → Webview sends { type: 'permission:response', decision, toolId }
    → Extension host receives response
    → If 'always_allow' → update permission config to 'allow'
    → Execute or skip based on decision
```

## Files to Modify

### 1. New: `webview/src/components/AuthorizationDialog.vue`
- Modal dialog showing tool name, category, arguments preview
- Three buttons: "仅本次允许" (Allow Once), "总是允许" (Always Allow), "拒绝" (Deny)
- Shows tool icon and description for context

### 2. New: `src/core/permissions.ts`
- Permission category definitions
- Permission state management (in-memory + persisted)
- `checkPermission(toolName)` → 'allow' | 'ask' | 'deny'
- `setPermission(category, level)` → update config
- `getPermissionConfig()` → current settings

### 3. Modify: `src/agent/engine.ts`
- In `executeToolCalls()`: add permission check before each `tool.execute()`
- If 'ask': pause execution, send request to webview, wait for response
- Add `permissionRequest` and `permissionResponse` event handling
- Store `pendingPermissionResolve` callback for async resolution

### 4. Modify: `webview/src/components/ConfigPanel.vue`
- Add "权限控制" section with toggle switches for each category
- Options: 允许 / 询问 / 拒绝

### 5. Modify: `webview/src/composables/useConfig.ts`
- Add permission config state
- Handle `permissionConfig` messages

### 6. Modify: `webview/src/protocol.ts`
- Add permission-related message types

### 7. Modify: `src/vscode/panel.ts`
- Handle permission config save/load
- Forward permission requests between engine and webview
- Persist permission config to VS Code settings

## Permission Config Storage

```typescript
interface PermissionConfig {
  fileRead: 'allow' | 'ask' | 'deny'
  fileWrite: 'allow' | 'ask' | 'deny'
  terminal: 'allow' | 'ask' | 'deny'
  git: 'allow' | 'ask' | 'deny'
  diagnostics: 'allow' | 'ask' | 'deny'
  web: 'allow' | 'ask' | 'deny'
  task: 'allow' | 'ask' | 'deny'
  checkpoint: 'allow' | 'ask' | 'deny'
}
```

Default: File Read, Git, Diagnostics, Task, Checkpoint = `allow`; File Write, Terminal, Web = `ask`

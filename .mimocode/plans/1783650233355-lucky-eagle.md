# Session Management System - Complete Redesign

## Goal
Upgrade the current checkpoint/snapshot system to a full session management system with history, search, sharing, and auto-save.

## Current State

### What Exists
- **Checkpoints**: Git commits with file changes + metadata
- **Metadata**: `.vte-metadata.json` stores conversation state (messages, mode, tokenStats)
- **Chat History**: In-memory array in panel.ts, synced to webview on reconnect
- **UI**: CheckpointBar.vue with simple list, save/restore/delete actions

### Limitations
1. No persistent session list (only git log queries)
2. No session metadata (title, summary, tags)
3. No search functionality
4. No export/import
5. No auto-save
6. Metadata and file snapshots are separate commits

## Proposed Architecture

### Data Model

```typescript
// Session metadata stored in .vte/sessions/index.json
interface SessionIndex {
  sessions: SessionMeta[]
  version: number
}

interface SessionMeta {
  id: string                    // unique session ID
  name: string                  // user-defined or auto-generated
  summary?: string              // AI-generated summary
  tags: string[]                // user tags
  createdAt: number             // timestamp
  updatedAt: number             // timestamp
  messageCount: number          // number of messages
  model: string                 // model used
  tokenUsage: { prompt: number; completion: number }
  checkpointId?: string         // linked git commit hash
  thumbnail?: string            // first user message (truncated)
}

// Each session also has its own git commit in shadow-git
// Metadata stored in commit message or separate file
```

### File Structure

```
.vte/
├── checkpoints/
│   └── <workspaceHash>/
│       └── .git                    # shadow-git repo
└── sessions/
    ├── index.json                  # session list
    └── <sessionId>/
        ├── metadata.json           # session metadata
        └── messages.json           # full conversation history
```

## Features

### 1. Session List (Enhanced CheckpointBar)
- Show all sessions with name, date, message count, model
- Sort by date, name, or usage
- Filter by tags
- Search by content

### 2. Session Save
- Auto-save on every N messages (configurable)
- Manual save with custom name
- Generate summary using LLM (optional)

### 3. Session Restore
- Restore conversation history
- Restore file states (from git)
- Option to restore only files or only conversation

### 4. Session Share
- Export session as JSON file
- Import session from JSON
- Copy session link (future: cloud sync)

### 5. Session Search
- Full-text search across all sessions
- Filter by tags, date range, model
- Search within conversation content

## Implementation Plan

### Phase 1: Session Data Layer

**Files to create/modify:**
- `src/agent/session-manager.ts` - Core session management
- `src/agent/session-types.ts` - Type definitions

```typescript
// session-manager.ts
export class SessionManager {
  private sessionsDir: string
  private indexPath: string
  
  constructor(workspaceRoot: string)
  
  // CRUD operations
  async createSession(name?: string): Promise<SessionMeta>
  async getSession(id: string): Promise<SessionData | null>
  async listSessions(): Promise<SessionMeta[]>
  async updateSession(id: string, updates: Partial<SessionMeta>): Promise<void>
  async deleteSession(id: string): Promise<void>
  
  // Message operations
  async addMessage(sessionId: string, message: ChatMessage): Promise<void>
  async getMessages(sessionId: string): Promise<ChatMessage[]>
  
  // Search
  async searchSessions(query: string): Promise<SessionMeta[]>
  
  // Export/Import
  async exportSession(sessionId: string): Promise<string>
  async importSession(data: string): Promise<SessionMeta>
  
  // Auto-save
  async autoSave(sessionId: string, state: SessionState): Promise<void>
}
```
